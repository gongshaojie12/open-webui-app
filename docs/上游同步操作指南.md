# 上游仓库同步操作指南

## 背景

本项目是从 [cogwheel0/conduit](https://github.com/cogwheel0/conduit) fork 而来，在 `dev-0.0.1` 分支上进行了自定义开发（竹云登录、品牌改名等）。当上游仓库有更新时，需要将更新同步到本地，同时保留 `dev-0.0.1` 的自定义修改。

## 分支结构

```
cogwheel0/conduit (upstream)
    └── main ──────────────────── 上游主分支
         │
gongshaojie12/open-webui-app (origin)
    ├── main ──────────────────── 与上游保持同步
    └── dev-0.0.1 ─────────────── 自定义开发分支（从 main 创建）
```

## 前置条件

### 1. 添加上游远程仓库（仅首次需要）

```bash
git remote add upstream https://github.com/cogwheel0/conduit.git
```

### 2. 验证远程仓库配置

```bash
git remote -v
```

预期输出：

```
origin    https://github.com/gongshaojie12/open-webui-app.git (fetch)
origin    https://github.com/gongshaojie12/open-webui-app.git (push)
upstream  https://github.com/cogwheel0/conduit.git (fetch)
upstream  https://github.com/cogwheel0/conduit.git (push)
```

## 同步操作步骤

### 第一步：拉取上游最新代码

```bash
git fetch upstream
```

此命令会从上游仓库下载所有新的提交，但不会修改本地任何分支。

### 第二步：合并上游更新到本地 main

```bash
# 切换到 main 分支
git checkout main

# 将上游 main 合并到本地 main（通常是快进合并，不会有冲突）
git merge upstream/main
```

### 第三步：推送 main 到自己的 GitHub

```bash
git push origin main
```

### 第四步：合并 main 到 dev-0.0.1

```bash
# 切换到开发分支
git checkout dev-0.0.1

# 将更新后的 main 合并到 dev-0.0.1
git merge main
```

> **注意**：这一步可能会产生冲突，需要手动解决。常见冲突文件见下方说明。

### 第五步：解决冲突（如果有）

如果出现冲突：

```bash
# 查看冲突文件
git status

# 手动编辑冲突文件，解决冲突标记（<<<<<<< ======= >>>>>>>）

# 标记冲突已解决
git add <冲突文件>

# 完成合并
git commit
```

### 第六步：推送 dev-0.0.1 到 GitHub

```bash
git push origin dev-0.0.1
```

## 完整命令速查

```bash
# 1. 拉取上游
git fetch upstream

# 2. 同步 main
git checkout main
git merge upstream/main
git push origin main

# 3. 合并到开发分支
git checkout dev-0.0.1
git merge main
# （解决冲突，如果有的话）
git push origin dev-0.0.1
```

## 可能产生冲突的文件

以下文件在 `dev-0.0.1` 中有自定义修改，当上游也修改这些文件时可能产生冲突：

| 文件 | dev-0.0.1 修改内容 | 冲突处理建议 |
|------|-------------------|-------------|
| `lib/l10n/app_*.arb` (10个语言文件) | 品牌名 Conduit → 众小智AI | 保留品牌改名，同时接受上游新增的国际化字段 |
| `lib/core/services/api_service.dart` | 无直接修改，但依赖关系可能受影响 | 通常接受上游更新即可 |
| `lib/features/auth/views/authentication_page.dart` | 大量修改（竹云登录、去除功能） | 保留 dev-0.0.1 的版本，手动合入上游新功能 |
| `lib/features/auth/views/sso_auth_page.dart` | 添加 SSL 证书处理、自定义参数 | 保留 dev-0.0.1 的修改，合入上游改动 |
| `lib/core/router/app_router.dart` | 路由重定向逻辑修改 | 保留 dev-0.0.1 的路由逻辑 |
| `lib/core/providers/app_providers.dart` | 自动配置服务器 | 保留 dev-0.0.1 的自动配置逻辑 |
| `lib/core/models/backend_config.dart` | 添加 focusmedia OAuth 提供者 | 保留 dev-0.0.1 的添加 |
| `android/app/src/main/res/values/strings.xml` | 品牌名修改 | 保留 dev-0.0.1 的品牌名 |
| `ios/Runner/Info.plist` | 应用名修改 | 保留 dev-0.0.1 的应用名 |

## 注意事项

1. **合并前确保工作区干净**：如果有未提交的修改，先 `git stash` 暂存
2. **参考改动记录**：解决冲突时可参考 `docs/dev-0.0.1_升级改动记录.md`，了解 dev-0.0.1 的每处修改意图
3. **合并后测试**：完成合并后建议运行 `flutter analyze` 和 `flutter test` 确保代码正确
4. **避免在 main 上直接开发**：main 分支应始终与上游保持同步，所有自定义开发在 dev-0.0.1 上进行
